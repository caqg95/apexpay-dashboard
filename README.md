## Descripción del Proyecto

ApexPay Dashboard es un proyecto desarrollado como parte de las prácticas del curso **Control de Versiones para Desarrollo Full Stack**. El objetivo es aplicar el uso de Git y GitHub para la gestión del código fuente, el trabajo colaborativo y el seguimiento de cambios mediante un flujo de trabajo basado en ramas, commits y pull requests.

---

# 3. Objetivos del Laboratorio

## Objetivo General

Aplicar los conceptos fundamentales de control de versiones utilizando Git y GitHub en un entorno de desarrollo colaborativo.

## Objetivos Específicos

- Crear y administrar un repositorio remoto en GitHub.
- Gestionar el código fuente mediante Git.
- Implementar un flujo de trabajo colaborativo utilizando ramas.
- Registrar cambios mediante commits descriptivos.
- Integrar cambios mediante Pull Requests y procesos de Merge.
- Mantener la trazabilidad de las modificaciones realizadas durante el desarrollo.

---

# 4. Conclusiones

1. Git permite mantener un historial detallado de los cambios realizados en un proyecto de software.
2. GitHub facilita el trabajo colaborativo mediante herramientas como ramas, Pull Requests y revisiones de código.
3. El uso de buenas prácticas de control de versiones mejora la organización y calidad del desarrollo.
4. La integración de cambios mediante Pull Requests ayuda a reducir errores y mantener la estabilidad del proyecto.
5. El trabajo en equipo utilizando herramientas de control de versiones es una habilidad fundamental en el desarrollo moderno de software.
